# 雾霾探测系统设计实验报告

## 一、问题描述

雾霾会影响出行安全和人体健康，因此需要在出行前查看当前城市的天气情况、空气质量指数和主要污染物指标。本系统设计一款手机端雾霾探测应用，实现城市定位、服务器端保存、天气详情查询、空气质量指数展示和温湿度趋势显示。

## 二、方案设计

系统采用“Android 手机端 + Spring Boot 服务端”的架构。手机端使用原生 Android Java 开发，调用手机系统定位功能获取经纬度；服务端使用 Maven 管理 Spring Boot 项目，提供定位保存、百度逆地理编码、天气详情和空气质量指数接口。

总体流程如下：

1. 客户端进入页面后请求 `/api/location`，读取服务器端保存的城市。
2. 用户点击“手机定位”后，Android 应用申请定位权限，并通过 GPS 或网络定位获取经纬度。
3. 手机端调用 `/api/location` 上传经纬度，服务端通过百度逆地理编码解析城市、区县和行政区划代码，并保存到服务器端。
4. 手机端调用 `/api/weather?city=城市名` 获取天气详情、AQI、污染物和趋势数据。
5. 手机端把返回数据渲染到天气卡片、AQI 卡片、污染物指标和折线图。

Spring Boot 后端保存文件为 `maven-server/data/store.properties`，其中包含当前定位城市、区县、展示名称、坐标、行政区划代码和更新时间，满足“定位城市保存在服务器端”的要求；天气和空气质量结果也会在服务端生成并保存最近一次记录。

为了让多个 App 和同一个 App 的多台手机共用同一个服务端，服务端采用类似 Cookie 的客户端 session 机制。客户端第一次请求时如果没有 session，或者 session 已过期/无效，服务端会创建随机 UUID session，通过 `X-Client-Session-Id` 响应头和 `Haze-Session` Cookie 返回给客户端。Android 端把 session 保存到本机 `SharedPreferences`，后续请求继续带回。session 默认 30 天有效，每次有效请求都会续期。服务端按 session 选择独立的定位存储文件，例如 `data/store-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.properties`。

## 三、数据获取

天气和空气质量数据优先通过百度地图开放平台接口获取。定位阶段调用百度逆地理编码接口，根据 WGS84 经纬度解析 `city`、`district` 和 `adcode`；天气阶段调用百度国内天气查询接口：

```text
https://api.map.baidu.com/weather/v1/?district_id=行政区划代码&data_type=all&ak=你的AK
```

服务端获取的数据包括温度、体感温度、相对湿度、风速、天气现象、PM2.5、PM10、CO、NO₂、SO₂、O₃ 和 AQI。如果 AK 在百度控制台启用了 `sn` 签名校验，服务端会使用配置的 SK 计算签名，SK 不会出现在请求 URL 中。

为了保证实验演示稳定性，当未配置 AK、未解析到行政区划代码或百度接口响应失败时，服务端会根据城市名称和当前时间生成合理的本地模拟数据，并在响应中标记数据来源为“Spring Boot 服务端模拟数据”。这样即使实验室网络受限，定位、保存、展示和图表功能仍能完整验证。

AQI 分级规则：

| AQI 范围 | 等级 | 建议 |
| --- | --- | --- |
| 0-50 | 优 | 适合户外活动 |
| 51-100 | 良 | 敏感人群减少长时间户外运动 |
| 101-150 | 轻度污染 | 敏感人群建议佩戴口罩 |
| 151-200 | 中度污染 | 儿童与老人减少外出 |
| 201-300 | 重度污染 | 尽量留在室内 |
| 300 以上 | 严重污染 | 避免外出 |

## 四、结果展示及分析

Android 应用首页由定位区域、服务器地址输入区、天气概览区域、空气质量区域、污染物监测区域和趋势图区域组成，使用原生 Android 控件与自定义折线图视图实现，适配手机屏幕展示。

实现结果：

1. 定位功能：用户点击“手机定位”后，城市和区县信息会显示在客户端，并保存到服务器端 `data/store.properties`。
2. 手动查询：用户可输入城市名称查询天气，查询不会写入 `/api/location`，也不会覆盖当前手机定位位置。
3. 天气详情：显示天气现象、温度、体感温度、湿度和风速。
4. 空气质量：显示 AQI、空气质量等级、PM2.5、PM10、CO、NO₂、SO₂、O₃ 和出行建议。
5. 动态图表：使用 Android 自定义 View 绘制最近时段温度、湿度和 AQI 折线图，直观展示变化趋势。

从测试角度看，系统接口清晰、功能闭环完整。服务器端保存了定位与天气数据，Android 前端能够动态读取并展示，符合实验题目对定位、界面设计、天气详情和空气质量指数的要求。

## 五、心得与体会

本实验重点在于把定位、服务器端数据保存、天气空气质量数据获取和移动端展示整合为一个完整系统。开发过程中需要考虑外部 API 不稳定、百度 AK/SK 签名、手机定位权限、手机屏幕适配和数据可视化等问题。通过加入本地模拟兜底数据，系统在无网络环境下仍能演示核心功能，提高了实验测试的可靠性。

## 六、运行与测试

服务端运行命令：

```bash
cd maven-server
mvn spring-boot:run
```

Android APK 打包方式：

```text
使用 Android Studio 打开 android/，执行 Build APK(s)
```
